Quickstart: podman pods

Installation und Start

Kopiere folgende Dateien in ein lokales Verzeichnis

  • create-playground.sh

  • env.playground

env.playground enthält 2 Secrets, die man vor dem Einsatz ändern sollte. Die Datei enthält eine kurze Anweisung dazu.
Pod erzeugen
bash create-playground.sh

Das Skript erzeugt einen Pod mit dem Namen play-titracka.

Pod starten
podman pod start play-titracka

Webzugriff

  • http://localhost:3000

  • login: admin

  • password: admin9999

Passwort für admin beim ersten Login ändern!

Erste Schritte

Podman pod: nützliche Kommandos

Pod anzeigen
podman pod ls
Pod stoppen
podman pod stop play-titracka
Welche Container laufen
podman ps
Pod löschen
podman pod stop play-titracka
podman pod rm play-titracka
podman pod rm löscht nicht die Volumes. Diese müssen separat gelöscht werden. Die Volumes aus dem Create-Skript beginnen alle mit play: 
local       play-titracka-postgres
local       play-titracka-redis
local       play-titracka-storage

Gelöscht wird ein Volumen mit podman rm <volume>.

Current Issues

  • Beim ersten Start muss man ggf. den Pod zweimal starten, da PostgreSQL beim Setup manchmal zu lange braucht. Solange das Volume dazu nicht gelöscht wird, tritt das Problem später nicht wieder auf.
